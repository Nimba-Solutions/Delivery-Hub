# Delivery Hub Changes — April 1, 2026

## 1. Orphaned Work Log Prevention (Code Change)

**Problem:** Work logs created without a parent WorkItemId (via Data Loader, anonymous Apex, etc.) are excluded from invoices and reports. This caused a 35.5h gap in Glen's March invoice (82h shown instead of 117.5h).

**Root Cause:** The LWC controller and REST API both validate WorkItemId, but the trigger handler did not block direct DML inserts without a parent.

**Fix:** Added validation in `DeliveryWorkLogTriggerHandler.onBeforeInsert()` that calls `addError()` if `WorkItemId__c` is null.

**Files Changed:**
- `force-app/main/default/classes/DeliveryWorkLogTriggerHandler.cls` — Added null check for WorkItemId__c before lockout enforcement
- `force-app/main/default/classes/DeliveryWorkLogTriggerHandlerTest.cls` — Added `testOrphanWorkLogBlocked()` test

**Data Fix (both orgs):**
- 10 orphaned work logs in Nimba Sandbox assigned to correct parent work items
- 10 orphaned work logs in MF Prod assigned to correct parent work items
- Resource title updated from null/Glen Bradford to "Sr SFDC Developer" on all affected logs
- Mapping:
