# Phase F — Billing / get paid + reporting (Delivery Hub, `delivery__`)
> Grounded read-only audit, 2026-06-18. THE #1 PAIN LIVES HERE: "every time I invoice, it's missing synced items it should have billed." Root cause is structural, not one bug — the invoice is a **frozen, SHA-256-hashed snapshot taken at generation** (day-1 for Monthly), so any worklog that relays/syncs/approves/requeues *after* generation is permanently absent until manual regeneration.

## 1. AS-BUILT FLOW
**Worklog → outbound relay:** `WorkLog__c` (`HoursLoggedNumber__c`, `WorkDateDate__c`, `WorkItemLookup__c`). `DeliveryWorkLogTriggerHandler.onBeforeInsert` **hard-blocks logs with no parent WI**, defaults null date to today, enforces month-end lockout, runs cap enforcement. `onAfterInsert/Update` apply the approval gate (`RequireWorkLogApprovalDateTime__c` set → only Approved/null sync; Draft syncs on approval). Eligible → `DeliveryWorkLogRelayMinter.mintForLogs` (`without sharing` — the nimba 65-row/138.5h guest loss fix), minting `SyncItem__c` (Queued/Outbound) in **Hub mode** (WI's `ClientNetworkEntityLookup__r` Client/Both + endpoint) or **Push mode** (WI fans out via active `WorkRequest__c` → Vendor/Both + endpoint). GlobalSourceId inherited (multi-hop dedup). `DeliverySyncItemProcessor` drains Queued, callout, Synced/Failed.
**Invoice snapshot:** `DeliveryDocGenerationService.generateDocument` → `buildSnapshot` queries **in the generating org only**: entity, issuer, WIs where `ClientNetworkEntityLookup__c=entity`, worklogs, requests. **Worklog selection:** `WorkLog__c WHERE WorkItemLookup__c IN :itemIds AND WorkDateDate__c BETWEEN periodStart AND periodEnd AND (StatusPk__c='Approved' OR null)`. `serializeWorkItems` **skips WIs with `TotalLoggedHoursSum__c<=0` for Invoices**. Cost = rate ladder (WorkRequest `HourlyRateCurrency__c` > item `BillableRateCurrency__c` > entity `DefaultHourlyRateCurrency__c`). Snapshot → `SnapshotTxt__c`, **SHA-256 `DocumentHashTxt__c`**, capped 131072 chars (trims descriptions, never hours/rates). Prior balance + cumulative amountDue merged into `MetadataTxt__c`. Created Draft, version-stamped (supersedes prior same entity/period/template), random `PublicTokenTxt__c`. **Never updated after insert.**
**Send → A/R → payment:** `DeliveryDocEmailService.sendDocumentEmail` renders `DeliveryDocumentPdf` VF → PDF, emails from verified OWEA, sets Sent + `SentDateTime__c` + **only here sets `DueDateDate__c`** (parses `TermsTxt__c` "Net 30"). Portal view flips Sent→Viewed. `DeliveryDocApprovalService`: Approve → Approved + Approval txn (sign-required → Awaiting_Signatures); Dispute → Disputed + reason. `DeliveryDocPaymentService.recordPayment` inserts Payment `DeliveryTransaction__c`, **auto-flips Paid when `getTotalPaid >= TotalCurrency__c` (this invoice only)**. `DeliveryStripePaymentHandler.handle` maps `charge.succeeded`/`payment_intent.succeeded` → recordPayment, keyed on `metadata.document_id`.
**Reporting/notifications/Watcher:** `markOverdueInvoices` (Sent/Viewed past-due → Overdue); `DeliveryOverdueReminderService` (escalating 1/7/14/30, 24h cooldown); `DeliveryNotificationService` (in-app bells — **no billing-event notifications**); `DeliveryWatcherService.runAndReturnSummary` (daily, gated; real signals SLABreach/StuckStage/**ARAging**; stubs PaymentOps/FailedSyncTrend/UnhappySignal/UpcomingSignoff return empty). AR Aging = Invoice docs Overdue AND `DueDateDate__c < TODAY - minDaysOverdue` (default 7), top 5.

## 2. USER STEPS
**Dev:** log `WorkLog__c`; if approval on, approver flips Approved → relay mints. **Admin (invoice):** pick entity + Invoice template + period → Generate (Draft) → review → Send (Sent + DueDate). **Auto-billing:** set `EnableBillingDateTime__c` + `BillingFrequencyPk__c` + flip `EnableInvoiceGenerationDateTime__c` (Monthly generates day 1). **Client (portal):** open token URL → Approve / Dispute(reason) / Pay (Stripe). **Get paid:** recordPayment manual or Stripe auto; overdue scheduler chases; AR digest; triage failed relays in `deliverySyncRetryPanel`. **Recover missing:** `DeliverySyncReconciler.reconcileAll()` → retry Failed → **then regenerate the invoice**.

## 3. GENERALIZABLE vs MF-SPECIFIC
Generalizable: snapshot/hash/version/supersede model; worklog-selection + rate ladder; Net-N parsing; OWEA send with DKIM-aware error; portal approve/dispute/pay; transaction ledger + auto-Paid; prior-balance math; overdue + reminders; Watcher + AR aging; relay/reconcile/retry; Stripe handler. MF/config-dependent: which org generates (MF-Prod = billing org, relays must land first); rates populated (zero rate → silent $0 line); Vendor NE existing (else "Your Organization"); verified OWEA + DKIM; Stripe creds + `metadata.document_id` in the Checkout template; Watcher flags + recipients. The 65-row/138.5h and 69.5h incidents were data/config drops (code fixes shipped).

## 4. GAPS / CAVEATS (honest)
**Selection predicate:** worklogs in the generating org only, against a frozen snapshot taken at gen time.
**WHERE a billable item can be missing (the #1 pain):**
1. **🔴 Frozen-snapshot + late arrival (root cause)** — Monthly auto-gen fires day-1; ANY worklog that relays/syncs/approves/requeues *after* gen is permanently absent until regeneration.
2. **🔴 Un-minted relay** — WI with no Client entity AND no active WorkRequest bridge with an endpoint → `mintForLogs` no-ops → log never leaves the source org.
3. **🔴 Stuck Failed SyncItem** — failed callout sits Failed, does NOT auto-retry; needs manual panel retry or reconciler. `retryFailed` is Outbound-only.
4. **🔴 Reconciler Hub-mode blind spot** — `fillWorkLogGaps` only checks WorkLogs reachable via an active **WorkRequest** bridge; Hub-mode-only (Client-linked, no WorkRequest) logs are NOT reconciled.
5. **🟠 Status filter** — approval-on + still Draft at gen → excluded; null status billed (legacy double-edge).
6. **🟠 Zero-hours skip** — WI with stale `TotalLoggedHoursSum__c<=0` drops the line even if real logs exist.
7. **🟠 Period boundary** — `WorkDateDate__c` is the cut; null date auto-set to *today* → back-dated late entries miss their real period.
8. **🟠 Wrong/missing entity link** — keyed on `WI.ClientNetworkEntityLookup__c`.
**Double-count (mirror):** snapshot just sums logs with no dedup; if GlobalSourceId dedup fails, a relayed log lands twice → double-billed.
**Payment gaps:** 🔴 Stripe has **no idempotency on chargeId** (duplicate webhook → second txn); 🔴 **silent unlink** if no `metadata.document_id` (money in Stripe, A/R open); 🟠 **cumulative-vs-per-doc mismatch** (email shows invoice+prior, recordPayment marks Paid on this doc only → over-credit); no partial-payment status, no credit-memo, no refund/chargeback.
**DueDate/aging holes:** 🟠 DueDate set only on send → generated-but-unsent invoice (null DueDate) never overdues, never in AR aging; 🔴 **Approved-unpaid blind hole** — portal Approve → status Approved (neither Sent/Viewed nor Overdue) → never ages, never reminds, never in AR digest.
**Watcher stubs:** 🔴 the two signals that would catch the core pain — `WatcherFailedSyncTrendQueryService` + `WatcherPaymentOpsQueryService` — are empty stubs. **No detector compares "approved worklogs in the period" vs "worklogs on the invoice."**

## 5. PROS / CONS + ENHANCEMENTS (unscoped)
Pros: tamper-evident frozen snapshots; clean rate ladder; self-healing reconciler + retry UI; gated Watcher with audit rows; DKIM-aware send errors; multi-hop dedup; cumulative A/R math. Cons: frozen snapshot + day-1 gen = late items lost; no pre-send completeness check; Hub-only logs unreconciled; Failed relays need manual retry; Stripe non-idempotent + silent-unlink; Approved-unpaid never ages; cumulative-vs-per-doc mismatch; key detector signals are stubs.
Enhancements: **missing-billable detector (keystone)** — pre-gen/pre-send reconcile listing approved-in-period logs with no relay landed, stuck-Failed for the period, Draft-when-approval-required, Hub-only orphans, entity-unlinked WIs → block/warn before send; regenerate-on-late-arrival or supplemental-invoice path; Stripe idempotency key on chargeId + alert on payment-with-no-document_id; set DueDate at generation; add Approved to overdue/aging eligibility; partial-payment + credit-memo statuses; build FailedSyncTrend + PaymentOps signals; invoice sent/paid/disputed notifications; dedup guard (group logs by GlobalSourceId) at snapshot assembly.

## 7. AGENT OPERATING INSTRUCTIONS (instruction-engine)
**Mission:** assemble each period's invoice from approved+delivered worklogs, **detect missing/uncaptured billable items BEFORE send**, route to human for approval.
**Reads (decide nothing yet):** for entity+period in the billing org, approved worklogs already in snapshot scope; cross-org completeness inputs in each source org (approved logs in period; their SyncItem status; stuck Failed in period; Draft-when-approval-required; WIs with logs but no entity link; Hub-only logs with no WorkRequest = reconciler blind spot); existing invoice for entity+period; DueDate-null + Approved-unpaid invoices.
**Decides (the missing-item gate — keystone):** delta = (approved-in-period logs across all orgs) − (logs that will land in the snapshot). Classify each absent item: un-minted relay → automatable (mint/reconcile); stuck Failed outbound → automatable (requeue then reconcile); Hub-only orphan → **human** (reconciler won't heal); entity-unlinked WI → **human** (data fix); Draft-when-approval-required → **human** (approval is human); possible double-count → **human** (don't auto-delete money rows).
**Writes (safe/reversible only):** requeue Failed outbound SyncItems; enqueue `DeliverySyncReconciler`. **Do NOT** auto-generate, auto-send, auto-record payment, auto-delete dupes, or mutate a frozen snapshot.
**Routes to human:** present the completeness report ("N approved hours not captured by class, M possible dupes, P approved-unpaid aging silently") → explicit approval before generate, before send, and for any data fix; after human-approved gen, re-run the delta to confirm zero un-captured, then route the Draft for send approval.
**Errors:** snapshot-cap diagnostic, send/DKIM failure, Stripe terminal (missing document_id) = hard stops → human. Never mark Sent/Paid on an exception path.
**Automatable today:** detect the delta (read-only), requeue Failed outbound, run reconciler, produce the completeness report, flag DueDate-null / Approved-unpaid / Stripe-unlinked / duplicate-charge anomalies. **Needs human:** approve worklogs, link WIs to entities, resolve Hub-only orphans, dedupe money rows, authorize generate/send/payment, regenerate after late arrivals.
