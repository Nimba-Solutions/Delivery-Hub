# Phase C — Org install · cross-org connection · sandbox sync · team access (Delivery Hub, `delivery__`)
> Grounded read-only audit, 2026-06-18. The cross-org connection is real and robust at the *engine* level, but *setup* is half-manual: API keys aren't auto-minted, receiver approval is a manual queue, push-vs-poll hinges on endpoint config, and sandbox sync + post-refresh re-init are NOT turnkey.

## 1. AS-BUILT FLOW
**Install (zero-touch):** `DeliveryHubInstallHandler` (`InstallHandler`) runs every install/upgrade: `initializeDefaultSettings` (org-default settings, all flags null=OFF except ActivityLogging+BoardMetrics) → `enqueueInstallSeed` (Queueable seeds Feature catalog + DatasetTemplates in a fresh txn — postInstall seeding silently wrote 0 rows) → `syncLegacySettingsToFeatures` → `scheduleSyncJobs` (`DeliveryHubScheduler.scheduleAll`) → `backfillUserPermissionSets` → `backfillWorkItemStatusDefaults` → `enqueueInitialReconciliation`. Every step try/caught so install never fails on a side effect.
**Cron:** `DeliveryHubScheduler.scheduleAll` (idempotent, aborts-then-registers) creates 4 CronTriggers `Delivery Hub Sync - :00/:15/:30/:45`. Each tick runs ~17 sub-jobs (requeue Failed/Staged/Pending, escalations, recurring, `@future` poll → `DeliveryHubPoller`, digests, invoices, reminders, reconciliation [1 GMT hour], ETA recalc, log cleanup, auto-diagnose). Most are DateTime-flag-gated, no-op until opt-in.
**Handshake:** `DeliveryHubSetupController` drives the Setup LWC. `getSetupStatus` reads mothership endpoint from `CloudNimbusGlobalSettings__mdt.Default.EndpointUrlTxt__c` (=`https://cloudnimbusllc.com/api/delivery-hub/v1`). `checkPrerequisites` verifies a Site named `Delivery_Hub` (Active) + its guest holds `DeliveryHubGuestUser` + Remote Site reachable. `prepareLocalEntity` upserts a "Cloud Nimbus LLC" `NetworkEntity__c`. `performHandshake` generates a 256-bit shared key, POSTs a NetworkEntity packet to `{endpoint}/sync/NetworkEntity__c`, on 2xx self-stamps `ConnectionStatusPk__c='Connected'` + RemoteExternalId/OrgId. **Asymmetric:** initiator self-marks Connected; receiver lands `Pending Approval` (`DeliverySyncItemIngestor` line 470) → admin `approveConnection`/`rejectConnection`.
**Status lifecycle** (`ConnectionStatusPk__c`, restricted, default Pending): Pending → Pending Approval → Connected → Disabled. Separate `StatusPk__c` (Active/Inactive) — both must align (poller keys on Active; processor/reconciler on Connected).
**Outbound:** `DeliverySyncEngine.captureChanges` (from triggers) computes GlobalSourceId, fans out — Push (WorkRequest→Vendor/Both, endpoint set → Queued) + Hub (WorkItem→Client/Both; `EnableVendorPushDateTime__c`+endpoint → Queued real-time, else **Staged** parked for pull). `DeliverySyncItemProcessor` (Queueable) drains Queued, resolves endpoint, injects `X-Api-Key`/`X-Signature`/`X-Global-Source-Id`, POSTs, marks Synced + maps `processedId` back. **No endpoint → Staged (not Failed)**; ≥400 → Failed + retry++.
**Inbound:** `DeliveryHubSyncService` (`@RestResource /deliveryhub/v1/sync/*`, `global without sharing`): `@HttpPost` = opt-in key → opt-in rate limit → opt-in HMAC → 300s echo-suppression → Pending-Approval gate (403 except NetworkEntity) → `DeliverySyncItemIngestor.processInboundItem`. `@HttpGet /sync/changes?since=&clientId=` = poll (Staged scoped by clientId, flips Staged→Synced); `/sync/health`. `DeliveryHubPoller.pollUpdates` = client side of poll. Ingestor (~1290 lines): ledger resolution, GlobalSourceId cross-path dedup, WorkLog content-dedup (billing safety), child-before-parent Pending queue + self-heal, dual-FK translation, auto-parenting to sender Client entity, savepoint atomicity.
**Reconciler:** `DeliverySyncReconciler` (daily 6 GMT) scans Connected Vendor/Both, fills missing-outbound gaps with Queued items (covered = Synced/Queued/Processing/Staged). **Health/Repair:** `DeliveryHubHealthService.runAllChecks` (10 cards); `DeliveryHubRepairService.runRepair` (scheduleAll / requeueFailedSyncs / backfillPermSets / reconcileNow).

## 2. USER STEPS
1. **Install** 2GP pkg (`cci flow run install`); choose Install for All Users. 2. **Assign permsets** `DeliveryHubApp` (users), `DeliveryHubAdmin_App` (admins), `DeliveryHubGuestUser` (Site guest), `Delivery_Hub_Gantt_Fullscreen` (optional). 3. **Connect:** Delivery Hub Admin app → Quickstart Connection wizard (needs Site `Delivery_Hub` Active + guest permset + Remote Site `Delivery_Hub_Mothership`). For org-to-org peering: manually create matching `NetworkEntity__c` on both sides (endpoint, shared key/HMAC, OrgId, RemoteExternalId, EntityType, Active+Connected) + Remote Site; receiver approves the inbound entity. 4. **Sandbox sync (owner):** NOT one-button — manually create a `NetworkEntity__c` on the sandbox pointing at the source org's `/services/apexrest/delivery/.../sync` with shared key; sandbox poller pulls Staged items. 5. **Team deploy/test:** `pip install cumulusci` → `cci flow run dev_org --org dev` (non-namespaced scratch) → `cci org browser dev`; per-PR CI spins a namespaced scratch. **Deploy LWC via `cci task run deploy`** (handles `%%%CURRENT_USER%%%` tokens), never raw `sf project deploy`.

## 3. GENERALIZABLE vs MF-SPECIFIC
Generalizable: entire sync engine (Engine/Processor/Ingestor/Poller/Reconciler/SyncService), HMAC+key auth, echo suppression, Pending-Approval gate, ledger dedup, NetworkEntity schema + lifecycle, opt-in DateTime gates, install handler, scheduler, health/repair, multi-tenant poll isolation via clientId, overridable sync-field list (`CloudNimbusGlobalSettings__mdt.SyncFieldsJsonTxt__c`), CumulusCI dev flow. MF/Cloud-Nimbus-specific (config): the mothership endpoint + name + Remote Site hardcoded to cloudnimbusllc.com; the MF↔nimba↔dh-prod relay topology + NetworkEntity records are org data.

## 4. GAPS / CAVEATS (honest)
1. **🔴 API key NOT auto-generated** despite the field claiming "auto-generated on create" — only `performHandshake` mints one; for manually-created peers (sandbox + org-peering), an admin must generate + paste keys on both sides by hand. No UI button.
2. **🔴 Handshake approval asymmetric + half-manual** — initiator self-Connected; receiver must approve in a queue; never approved → all non-NetworkEntity inbound silently 403'd.
3. **🔴 Push-vs-poll endpoint config = #1 footgun** — blank endpoint → items park Staged forever until `requeueStagedItemsWithEndpoint` finds an endpoint. Real multi-day stalls documented. The two status fields (`StatusPk__c` Active vs `ConnectionStatusPk__c` Connected) must both be right.
4. **🔴 Sandbox sync NOT turnkey** — no clone feature; it's the generic engine pointed manually. **Sandbox refresh wipes NetworkEntity config + CronTriggers** (install handler doesn't run on refresh) — must re-run `scheduleAll` + re-create entities + re-activate Site/Remote-Site after each refresh.
5. **🟠 Team deploy only via CumulusCI** — namespace tokens break raw sfdx LWC deploys.
6. **🟠 Mothership endpoint + Site name hardcoded** in metadata; Site must be named exactly `Delivery_Hub`.
7. **🟠 Reconciler heals downstream (Vendor/Both) only** — pure upstream/Client-poll drift relies on the watermark, which can over-advance and skip.
8. **🟠 HMAC/rate-limit/API-key all opt-in** — a NetworkEntity with no key/secret accepts unsigned, unauthenticated inbound (backward-compat open access). Surface is `without sharing` as guest.

## 5. PROS / CONS + ENHANCEMENTS (unscoped)
Pros: native (no middleware), bidirectional with echo suppression + GlobalSourceId traceability, robust self-heal, multi-tenant isolation, safe-by-default install, hash-chained audit, generic health/repair. Cons: manual key exchange; asymmetric approval; dual-status confusion; silent Staged; sandbox-refresh fragility; hardcoded mothership/Site; CCI-only deploy; open-by-default if secrets unset; downstream-only reconciliation.
Enhancements: auto-mint key/HMAC in the NetworkEntity before-insert trigger; one-click "connect peer org" wizard (both-side entities + RSS + keys); surface Staged-with-no-endpoint as a health card; collapse/validate the two status fields; post-refresh "Repair Sandbox" button; configurable mothership endpoint (white-label); a `cumulusci.yml` sandbox-seeding task; upstream-poll reconciliation parity + watermark integrity check.

## 6. COWORK TEST SCENARIOS (exhaustive — 84; Pre→Action→Expected→Verify). `delivery__` prefix on subscriber orgs.
**Install/baseline (1–6):** fresh-install seeds settings (ActivityLogging+BoardMetrics non-null, notifications null); 4 cron jobs present; Feature/DatasetTemplate seeded async; re-install preserves toggled flags; permset backfill covers active users; health all-green baseline.
**Permsets (7–10):** assign Admin PSG via CCI; guest permset when Site exists; guest no-op when Site missing; idempotent guest assignment.
**Prereqs/wizard (11–17):** checkPrerequisites all-pass; missing Remote Site / inactive Site detected; mothership self-detection; prepareLocalEntity creates mothership NE; performHandshake happy (Connected + 64-hex key + RemoteExternalId/OrgId + "Connection Established" WI); handshake failure → AuraHandledException.
**Status transitions (18–26):** inbound NE → Pending Approval; pending notification fires; approveConnection→Connected; approve rejects non-pending; rejectConnection→Disabled; pending count/list; Pending-Approval gate 403s data sync; NetworkEntity exempt from gate; Disabled-entity outbound behavior (document actual: processor excludes only Pending Approval).
**Key/HMAC (27–35):** no key = open access; valid key 200; invalid key 401; key valid but not Connected 401; HMAC pass; HMAC fail 401; HMAC skipped when no secret; outbound injects headers; rate limit 429 + Retry-After.
**Push (36–44):** push creates Queued item; processor delivers + closes bridge (RemoteWorkItemId, LastOutboundSync stamp); no-endpoint → Staged park; Staged self-heals after endpoint set; 400+ → Failed + retry++; Failed requeue under limit; retry cap respected; dedup against Staged (in-place payload update); empty-payload guard.
**Poll (45–53):** /sync/changes returns Staged + flips Synced; clientId isolation; missing clientId 400; since watermark filters; bad route 404; /sync/health; poller advances watermark; poller skips misconfigured vendor; no-active-vendor error.
**Ingestor/dedup/self-heal (54–64):** inbound WI insert + ledger; update via ledger; echo suppression window; echo expiry; cross-path dedup; WorkLog content-dedup (no double-billing); child-before-parent → Pending; Pending resolves after parent; blank-WI create rejected; predecisional gate; auto-parent to Client entity.
**Reconciler/health/repair (65–74):** reconciler fills WI gap; ignores covered; WorkLog gap with inherited GlobalSourceId; runRepair scheduleAll; requeueFailedSyncs; unknown key throws; health no-cron FAIL; stale-Failed WARN; API-key-missing WARN; settings-missing FAIL.
**Sandbox/team (75–80):** refresh wipes cron (→ rerun scheduleAll); NE config lost on refresh; manual sandbox-from-prod pull; CCI dev_org scratch; CCI deploy handles tokens (vs raw sf fails); per-PR namespaced scratch ≥75%.
**Errors (81–84):** empty payload 400; malformed JSON 500; DepthProbe branch; deleted-target update falls through to insert.

## 7. AGENT OPERATING INSTRUCTIONS (instruction-engine)
**Watches:** `SyncItem__c` by Status/Direction (Failed retry≥3, Staged "Endpoint not configured", Pending inbound, stale-Failed >7d); `NetworkEntity__c` (Pending-Approval count, Status/ConnectionStatus coherence, blank endpoint/key/secret, stale sync timestamps); CronTrigger count (<4 degraded, 0 dead); `DeliveryHubHealthService` cards; `DeliverySync__e` events. **Decides + writes (automatable):** 0/<4 cron → `runRepair('scheduleAll')`; large aged Failed backlog → `requeueFailedSyncs`; drift → `reconcileNow`; permset gaps → `backfillPermSets`. Staged-no-endpoint and Pending children self-heal — only flag if endpoint still blank or row stuck across many ticks. **Error rules:** distinguish Staged (benign/waiting) from Failed (action); 401/403 outbound = auth/approval → human; 429 = back off. **Routes to human:** API key/HMAC exchange (no auto-mint); approving a Pending-Approval connection (trust decision); pointing/repointing endpoints + creating peer entities; Site/Remote-Site/sandbox-refresh re-init beyond scheduleAll; changing mothership endpoint (metadata deploy); enabling consequential feature gates. **Net:** automate idempotent self-healing repairs (all via `DeliveryHubRepairService`); escalate anything touching credentials, trust/approval, endpoint topology, or org metadata.
