0) Mental Models — How Salesforce Thinks

Metadata-Driven Stack

Runtime compiles these layers on every transaction. Your job: pick the right layer.

Order of Execution (Save)

Exam zingers
  • Only before-save can set field values efficiently.
  • Assignment rules only at creation. Escalation is time-based.
  • Publish events after commit to avoid duplicates on rollback.

1) Security & Access — Profiles → Permission Sets → OWD → Sharing

Layered Security Pyramid

Profiles/PermSets decide what you can do; OWD/Sharing decide which records you can do it to.

Quick Matrices

Exam traps
  • Permission Sets cannot override lack of visibility from OWD/sharing.
  • Page layout ≠ security. Use FLS for true hide.
  • Record type availability is granted by Profile; it is not a security barrier.

2) Standard Data Model — Sales, Service, Marketing

Core CRM ERD (mini)

Lead converts → Account + Contact (+ Opportunity). OppLineItem bridges Opportunity ↔ Product2 via PricebookEntry.

Key Relationships & Limits

  • Master-Detail (tight): child inherits owner/sharing; parent delete cascades; enables roll-ups.
  • Lookup (loose): optional, no ownership inheritance.
  • Junction: two M-D to parent objects to model M:N (e.g., ProjectResource__c).
  • Record Types: max ~200/object; govern picklists, page layouts, business processes.
Sales vs Service quick cues
  • Sales: Lead → Opp; Products/Pricebooks; Forecasts; Quotes/Orders (optionally CPQ).
  • Service: Case lifecycle; Assignment/Escalation; Entitlements/Milestones; Knowledge; Omni.

3) Record Types & Business Processes

What Record Types Do

  • Control picklist subsets per context.
  • Select page layouts per profile × record type.
  • Bind to Business Processes (Lead Status, Case Status, Opp Stage sets).
Not security. Record types do not grant access; profiles/perm sets and sharing do.

Business Process Binding

Users only see record types their profile allows; defaults apply when only one choice.

4) Automation — Flow First, Approvals for Human Gates, Events for Decoupling

Decision Tree

  • Need to block save / set fields efficiently → Before-save Record-Triggered Flow.
  • Need related-record updates / emails after save → After-save Flow.
  • Human decision & record lock → Approval Process.
  • Multi-subscriber async reaction or integration → Platform Events (HVPE for scale).
  • Legacy-only: Workflow (outbound message), Process Builder (migrate to Flow).
Approval essentials
  • On submit: record locks; steps can be serial/parallel.
  • Actions: field updates, emails, tasks, outbound messages.

5) Sales Cloud — Lead→Opp, Products/Pricebooks, Forecasts

Lead Conversion

  • Maps Lead → Account + Contact + (optional) Opportunity.
  • Field mapping required; duplicates managed via dupe rules.
  • Converted leads are read-only; cannot “unconvert”.

Products & Pricing

  • Product2 (SKU via ProductCode), Pricebook (context), PricebookEntry (bridge).
  • OpportunityLineItem holds price/qty/discount (can use Schedules).

Forecasts & Quotes

  • Collaborative Forecasts: forecast types (revenue/qty/product/territory), categories (Pipeline/Best/Commit/Closed).
  • Quotes can sync to Opp; Orders represent commitments.
  • CPQ adds bundles, rules, subscriptions, proration.
Exam cue: Forecasting categories ≠ stages; mapping is configurable.

6) Service Cloud — Cases, Entitlements, Omni-Channel

Cases

  • Channels: Web-to-Case, Email-to-Case, Chat/Messaging, Phone, Community.
  • Assignment Rules at create; Escalation Rules time-based.
  • Case Teams vs Queues: collaboration vs ownership pool.

Entitlements & Milestones

  • Define SLA response/resolution clocks; timers can pause/resume.
  • Milestone actions on approaching/violation (notify, escalate).

Omni-Channel & Productivity

  • Routes work by priority/capacity/skills; prevents cherry-picking.
  • Macros automate agent clicks; Quick Text for snippets.

7) Reporting & Dashboards

Report Types & Formulas

  • Tabular / Summary / Matrix / Joined.
  • Custom Report Types define join paths; beware “with/without” relationships.
  • Row-level vs Summary formulas; Buckets; Cross filters (with subfilters).

Dashboards

  • Components, filters, Dynamic Dashboards (cannot be scheduled).
  • Historical trending & reporting snapshots for change-over-time.

8) Data Management & Governance

Data Ops

  • Import Wizard (simple), Data Loader (bulk CSV), API-based ETL.
  • Mass Transfer / Mass Delete (selected objects).
  • Duplicate & Matching Rules (alert vs block).

Scale & External

  • Big Objects for billions (read-only, indexed queries).
  • External Objects via Salesforce Connect (OData); data stays outside.
  • External Services (OpenAPI) → invocable actions in Flow.

Compliance & Protection

  • Shield Encryption (deterministic vs probabilistic), Key Management, BYOK.
  • Field History / Field Audit Trail; Event Monitoring.
  • Data Classification & Health Check; Optimizer.

9) Identity — My Domain, SSO, Certificates, Session Policies

My Domain

  • Org-specific login URL; enables Lightning, SSO, Experience Cloud branding.
  • Required before using modern components and SSO.

SSO & Certs

  • SAML/OIDC; JWT OAuth for server-to-server.
  • Certificates for signing/verification; rotate before expiry.

Session & Access Controls

  • Login hours & IPs (profile-level), MFA, Session Security Levels.
  • CORS, CSP Trusted Sites, Named/External Credentials for callouts.

10) Experience Cloud — External & Guest Access

Sharing Sets

Grant access to records related to a portal user’s Account/Contact (e.g., customers see their Cases/Orders).

Guest Users

Unauthenticated users with a highly constrained profile; avoid write access; audit public exposure.

11) Events — Platform Events, HVPE, CDC

When to Use What

  • CDC: replicate object CRUD changes externally; zero-code capture.
  • PE: business milestone events (internal/external subscribers).
  • HVPE: high throughput, large fan-out (paid add-on).
Publish after commit; include idempotency keys; keep payloads lean (IDs).

Inside vs Outside

  • In-org: After-save Flow/Apex or PE trigger; Queueables for heavy work.
  • External: Pub/Sub API (gRPC) with replay; Event Relay to cloud bus.

12) Environments & DevOps

Sandboxes

  • Dev/Dev Pro/Partial/Full; refresh cadence & templates.
  • Seeding & anonymization for test data.

Deployments

  • Change Sets (admin-friendly) vs SFDX (source-driven) vs DevOps Center.
  • Run tests in prod deployments; backout plan & post-deploy steps.

Packaging

  • Unlocked Packages for modular metadata; Managed for ISVs.
  • Permission Set Groups & Muting for access packaging.

13) Exam Strategy & Traps

High-Yield Topics

  • Security model (Profiles vs Perm Sets vs OWD vs Sharing vs Restriction).
  • Automation choices (Flow vs Approval vs Events; legacy trivia).
  • Reporting (report types, dynamic dashboards, formulas, cross filters).
  • Lead conversion, Sales vs Service quirks, Campaign Members.

Common Traps

  • Page layout is not security; FLS governs API/reports.
  • Assignment rules only on creation; Escalation is time-based.
  • Dynamic dashboards cannot be scheduled.
  • Record Types control picklists/layouts, not access.
  • Permission Sets add; cannot restrict.
Last-Week Checklist
  • Draw the security pyramid from memory.
  • Explain order of execution including where approvals/events sit.
  • Build a quick Flow: before-save (set fields), after-save (create related).
  • Create a custom report type with a cross filter and a summary formula.
  • Configure a case assignment rule + escalation rule + entitlement milestone.

Appendix: Glossary & Quick Definitions

Security

  • FLS: Field-level read/edit access.
  • OWD: Org baseline record visibility.
  • Restriction Rule: Post-filter visibility reducer.
  • Scoping Rule: Default UX filter.

Automation

  • Flow: Declarative logic engine (record/screen/scheduled).
  • Approval: Human gate + locking.
  • PE/HVPE: Publish/subscribe domain events; HVPE for scale.
  • CDC: Object change streams for replication.

Data

  • Big Object: Massive, read-optimized archive object.
  • External Object: Virtualized table via OData.
  • PricebookEntry: Bridge for Product2 pricing.
